# 📅 DevOps Day 01: Introduction to DevOps

### 1. What is DevOps?

DevOps is a **practice and organizational culture** that brings development and operations teams together. It removes traditional silos to increase a company's ability to deliver applications and services at maximum velocity.

- **Core Goal:** Improve and fasten application delivery.
- **Key Pillars of DevOps Culture:**
  - **Automation:** Replacing manual, repetitive tasks with automated scripts and pipelines.
  - **Quality:** Ensuring stable releases with robust code quality standards.
  - **Monitoring:** Tracking application health and system performance continuously.
  - **Testing:** Validating every single code change before it hits production.

> **Definition Summary:** DevOps is a continuous process of improving application/software delivery by emphasizing **Automation, Quality, Continuous Monitoring, and Continuous Testing**.

---

### 2. Why DevOps? (The Problem Statement)

In the traditional software model (10 years ago vs. today's 10-day agile cycles), delivering code from a developer's laptop to an actual production server was an operational bottleneck filled with heavy **manual effort**.

#### The Legacy Manual Workflow:

- **The Pipeline:** Developer ➔ Central Location ➔ Server / App Server ➔ Deployment ➔ Production ➔ Customers.
- **The Bottlenecks:**
  - Separate siloed roles: **System Administrators**, **Server Admins**, and **Build & Release (BRE) / SRE** engineers.
  - Handoffs between these teams were highly fragmented, requiring manual scripts, long coordinate delays, and extensive troubleshooting time.
